# CLAUDE.md

Guidance for Claude Code (claude.ai/code) when working in this repository.

## Overview

Laravel package that generates event sourcing domain structures for [Spatie's Laravel Event Sourcing](https://github.com/spatie/laravel-event-sourcing). It provides the Artisan command `make:event-sourcing-domain`, which scaffolds complete domain directories (events, projections, projectors, aggregates, reactors, actions, DTOs, notifications, and PHPUnit tests).

## Development Commands

```bash
composer test          # Run all PHPUnit tests
composer test-coverage # Coverage report + badge
composer fix           # Auto-fix code style (Laravel Pint)
composer check         # Check code style without fixing
composer static        # PHPStan / LaraStan static analysis
composer all           # test + fix + check + static
composer clear         # Clear testbench skeleton
composer prepare       # Discover packages
composer build         # Build testbench workbench
composer docs:sync     # Regenerate code-derived docs (compatibility, options, unsupported types)
composer docs:check    # Fail if generated docs are out of sync (CI gate)
```

## Conventions

- Aggregates are generated only when the primary key is `uuid`
- Failed events and notifications are opt-in via command flags
- Indentation is 4 spaces (the `--indentation` option affects generated code)
- User documentation lives in the Astro Starlight site under `website/` (published to GitHub Pages)
- Compatibility, command options and unsupported column types are **code-derived**: generated by `bin/docs-sync.php` from `composer.json`, `.github/workflows/test.yml`, the command `$signature` and `BlueprintUnsupportedInterface`. Never hand-edit `website/src/generated/` or the README compatibility block â€” run `composer docs:sync` after changing those sources (CI enforces it via `composer docs:check`)
