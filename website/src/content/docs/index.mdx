## Why this package

[Spatie's Laravel Event Sourcing](https://spatie.be/docs/laravel-event-sourcing) is powerful, but wiring up every event, projection, projector, aggregate and reactor by hand is repetitive. This package generates the whole domain structure — consistently, and optionally from an existing database migration — so you can focus on behaviour instead of boilerplate.

<CardGrid stagger>
  <Card title="Complete domains" icon="seti:folder">
    Events, projections, projectors, actions and DTOs every time; aggregates and reactors on demand.
  </Card>
  <Card title="From your migrations" icon="seti:db">
    Parse an existing migration and map every column into the DTO, projection and projector automatically.
  </Card>
  <Card title="Tests included" icon="approve-check">
    Generate working PHPUnit tests covering create / update / delete out of the box.

  </Card>

## Install

```bash
composer require albertoarena/laravel-event-sourcing-generator
```

Then generate your first domain:

```bash
php artisan make:event-sourcing-domain Animal --domain=Animal
```

See [Installation](/laravel-event-sourcing-generator/getting-started/installation/) for compatibility, and the [Quick start](/laravel-event-sourcing-generator/getting-started/quick-start/) for what gets generated.
